# 需求文档

## 介绍

Agent Platform 是一个基于 Web 的智能体对话创建平台。用户拥有三种核心资产：智能体（Agent）、Skill 和 MCP（Model Context Protocol）。这些资产可以发布到公开市场供其他用户使用。用户可以创建、管理、使用自己或他人公开的智能体进行对话交互。

平台提供完整的资产管理生命周期，包括创建、配置、发布、搜索、使用和版本管理。同时支持知识库集成，增强智能体的知识检索能力。

## 需求

### 需求 1：智能体管理

**用户故事：** 作为平台用户，我想要创建和管理我的智能体，以便根据业务需求定制 AI 助手。

#### 验收标准（EARS 格式）

1. 当用户进入"创建智能体入口"页面时，系统应展示智能体创建的引导流程。
2. 当用户点击"创建智能体"按钮时，系统应提供智能体基本信息配置表单，包括名称、描述、模型选择、最大步骤数。
3. 当用户完成基本信息填写时，系统应允许用户配置协作智能体和可用工具。
4. 当用户配置智能体工具时，系统应提供工具分组列表，支持刷新工具列表和参数配置。
5. 当用户完成智能体配置时，系统应保存智能体并提供调试功能进行测试验证。
6. 当用户在智能体列表页面时，系统应展示用户拥有的所有智能体卡片，包括名称、描述、状态和操作按钮。
7. 当用户点击智能体卡片的"编辑"按钮时，系统应跳转到智能体配置界面进行修改。
8. 当用户点击智能体卡片的"复制"按钮时，系统应创建智能体副本并添加到列表。
9. 当用户点击智能体卡片的"删除"按钮时，系统应弹出确认对话框，确认后删除智能体。
10. 当用户点击智能体卡片的"导出"按钮时，系统应生成 JSON 配置文件供下载。
11. 当用户导入智能体配置 JSON 文件时，系统应验证文件格式并创建智能体副本。

### 需求 2：智能体市场发布与发现

**用户故事：** 作为平台用户，我想要将我的智能体发布到公开市场，以便其他用户可以发现和使用我的智能体。

#### 验收标准（EARS 格式）

1. 当用户在智能体配置界面完成调试并验证智能体可用时，系统应提供"发布"按钮。
2. 当用户点击"发布"按钮时，系统应弹出发布确认对话框，包含发布范围（公开/私有）选项。
3. 当用户选择"公开发布"并确认时，系统应将智能体发布到公开市场，状态变更为"已发布"。
4. 当用户在智能体市场页面时，系统应展示所有公开发布的智能体列表，支持分类浏览。
5. 当用户在市场搜索框输入关键词时，系统应返回匹配的智能体结果。
6. 当用户点击市场智能体卡片时，系统应展示智能体详情，包括名称、描述、作者、工具列表和使用统计。
7. 当用户点击市场智能体的"使用"按钮时，系统应创建对话会话并启动该智能体。
8. 当用户点击市场智能体的"收藏"按钮时，系统应将智能体添加到用户收藏列表。
9. 如果智能体已被发布，则系统应支持版本管理，允许用户查看历史版本和回滚。
10. 当已发布智能体有更新时，系统应提示用户是否发布新版本。

### 需求 3：Skill 管理

**用户故事：** 作为平台用户，我想要创建和管理 Skill，以便定义可复用的能力和工作流程。

#### 验收标准（EARS 格式）

1. 当用户进入 Skill 管理页面时，系统应展示用户拥有的所有 Skill 卡片列表。
2. 当用户点击"创建 Skill"按钮时，系统应提供 Skill 创建表单，包括名称、描述、触发条件和能力定义。
3. 当用户填写 Skill 信息时，系统应提供 YAML/Markdown 格式编辑器用于定义 Skill 内容。
4. 当用户完成 Skill 创建时，系统应保存 Skill 并验证格式正确性。
5. 当用户点击 Skill 卡片的"编辑"按钮时，系统应打开 Skill 编辑界面。
6. 当用户点击 Skill 卡片的"删除"按钮时，系统应弹出确认对话框后删除。
7. 当用户点击 Skill 卡片的"导出"按钮时，系统应生成 Skill 配置文件。

### 需求 4：Skill 市场发布与发现

**用户故事：** 作为平台用户，我想要将 Skill 发布到市场，以便其他用户可以发现和应用我的 Skill。

#### 验收标准（EARS 格式）

1. 当 Skill 市场页面加载时，系统应展示所有公开发布的 Skill 列表，支持分类和搜索。
2. 当用户点击 Skill 市场卡片的"详情"按钮时，系统应展示 Skill 详细信息，包括描述、使用场景和示例。
3. 当用户点击 Skill 市场卡片的"导入"按钮时，系统应将 Skill 复制到用户资产列表。
4. 当用户在 Skill 管理页面点击"发布"按钮时，系统应将 Skill 发布到公开市场。
5. 如果 Skill 已发布到市场，则系统应展示使用次数和评分统计。
6. 当用户对已使用的 Skill 进行评价时，系统应记录评分并更新统计。

### 需求 5：MCP 管理

**用户故事：** 作为平台用户，我想要配置和管理 MCP 服务器，以便扩展智能体的工具能力。

#### 验收标准（EARS 格式）

1. 当用户进入 MCP 管理页面时，系统应展示用户配置的所有 MCP 服务器列表。
2. 当用户点击"添加 MCP"按钮时，系统应提供两种添加方式：URL 方式和 JSON 配置方式。
3. 当用户选择 URL 方式添加 MCP 时，系统应提供 Server 名称和 Server URL 输入框，支持 SSE 和 Streamable HTTP 协议。
4. 当用户选择 JSON 配置方式添加 MCP 时，系统应提供 JSON 编辑器和端口号输入框。
5. 当用户完成 MCP 配置时，系统应验证配置有效性并保存 MCP 服务器信息。
6. 当用户点击 MCP 卡片的"测试"按钮时，系统应测试 MCP 连接状态并返回结果。
7. 当用户点击 MCP 卡片的"删除"按钮时，系统应弹出确认后删除 MCP 配置。
8. 当用户点击 MCP 卡片的"启用/禁用"切换时，系统应更新 MCP 服务器的可用状态。
9. 如果 MCP 服务器连接失败，则系统应展示错误信息并提供重连选项。

### 需求 6：MCP 市场发布与发现

**用户故事：** 作为平台用户，我想要发现和使用公开的 MCP 服务，以便快速扩展智能体能力。

#### 验收标准（EARS 格式）

1. 当 MCP 市场页面加载时，系统应展示所有可用的公开 MCP 服务列表。
2. 当用户在 MCP 市场搜索框输入关键词时，系统应返回匹配的 MCP 结果。
3. 当用户点击 MCP 市场卡片的"详情"按钮时，系统应展示 MCP 详细信息，包括服务描述、提供的工具列表和使用说明。
4. 当用户点击 MCP 市场卡片的"接入"按钮时，系统应自动配置 MCP 并添加到用户资产列表。
5. 如果 MCP 需要特定参数配置，则系统应在接入后弹出参数配置对话框。
6. 当 MCP 服务有更新时，系统应提示用户更新版本。

### 需求 7：Web 对话界面

**用户故事：** 作为平台用户，我想要与智能体进行对话交互，以便完成任务并获得 AI 辅助。

#### 验收标准（EARS 格式）

1. 当用户进入对话页面时，系统应展示对话界面，包含消息区域、输入框和智能体选择器。
2. 当用户选择智能体时，系统应展示可用智能体列表，支持搜索和分类筛选。
3. 当用户在输入框输入消息并发送时，系统应将消息传递给选定的智能体并启动响应。
4. 当智能体响应时，系统应实时展示响应内容，支持流式输出显示。
5. 当智能体执行工具调用时，系统应展示工具执行状态和结果。
6. 当用户点击消息的"重新生成"按钮时，系统应重新请求智能体生成响应。
7. 当用户点击消息的"复制"按钮时，系统应复制消息内容到剪贴板。
8. 当用户点击"清空对话"按钮时，系统应弹出确认后清空当前对话历史。
9. 如果对话超过最大步骤数，则系统应提示用户并停止响应。
10. 当用户切换智能体时，系统应保留当前对话历史并切换智能体上下文。

### 需求 8：知识库管理

**用户故事：** 作为平台用户，我想要创建和管理知识库，以便为智能体提供领域知识检索能力。

#### 验收标准（EARS 格式）

1. 当用户进入知识库管理页面时，系统应展示用户拥有的所有知识库列表。
2. 当用户点击"创建知识库"按钮时，系统应提供知识库创建表单，包括名称、描述和索引配置。
3. 当用户上传文档到知识库时，系统应解析文档内容并建立索引。
4. 当用户点击知识库卡片的"管理文档"按钮时，系统应展示知识库中的所有文档列表。
5. 当用户删除知识库中的文档时，系统应更新索引并移除文档。
6. 当用户点击知识库卡片的"测试检索"按钮时，系统应提供检索测试界面，输入查询返回结果。
7. 当用户配置智能体时，系统应允许选择知识库检索工具并配置目标知识库。
8. 如果知识库索引失败，则系统应展示错误信息并支持重试。

### 需求 9：W3 OAuth 认证集成与权限

**用户故事：** 作为平台用户，我想要通过公司 W3 认证系统登录平台，以便无缝访问并管理我的资产。

#### 验收标准（EARS 格式）

1. 当用户首次访问平台时，系统应自动检测 W3 登录状态，未登录则跳转到 W3 OAuth 授权页面。
2. 当用户在 W3 OAuth 页面完成授权时，系统应接收 OAuth 回调，获取用户身份信息（用户ID、姓名、邮箱、组织等）。
3. 当用户通过 W3 OAuth 成功登录时，系统应生成平台会话令牌并加载用户资产。
4. 当用户会话过期时，系统应自动刷新令牌或重新引导用户通过 W3 OAuth 登录。
5. 当用户创建资产时，系统应自动设置资产所有者为当前 W3 用户。
6. 当用户设置资产权限时，系统应提供三种组内权限选项：同组可编辑、同组只读、私有。
7. 如果用户尝试访问无权限的资产，则系统应拒绝访问并提示权限不足。
8. 当用户退出登录时，系统应清除平台会话并跳转到 W3 登出页面。
9. 当管理员查看用户管理页面时，系统应展示从 W3 同步的用户列表和权限配置选项。
10. 当管理员修改用户权限时，系统应即时更新权限配置。

### 需求 10：资产管理统一入口

**用户故事：** 作为平台用户，我想要有一个统一入口管理所有资产，以便高效组织我的智能体、Skill 和 MCP。

#### 验收标准（EARS 格式）

1. 当侧边栏导航加载时，系统应展示三大资产类别：智能体、Skill、MCP。
2. 当用户点击导航菜单项时，系统应切换到对应的资产管理页面。
3. 当用户点击"创建"入口时，系统应展示创建引导页面，包含智能体、Skill、MCP 的创建选项。
4. 当创建引导页面展示时，系统应提供可视化卡片引导用户选择创建类型。
5. 当用户点击创建引导卡片的"开始创建"按钮时，系统应跳转到对应的创建表单页面。
6. 如果用户没有创建权限，则系统应隐藏创建入口并提示权限不足。
