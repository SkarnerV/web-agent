## 1. 设计原则与总体信息架构

### 1.1 产品定位

Agent Platform 是一个面向企业内部用户的"AI 资产工作台 + 对话工作区"。用户在这里创建、配置、发布三类核心资产（Agent / Skill / MCP），并通过对话界面使用它们。

### 1.2 设计原则

- **资产统一心智**：Agent / Skill / MCP 三者在列表页、市场页、详情页使用**完全一致的布局与交互范式**（卡片结构、按钮位置、发布流程），降低学习成本。
- **创建 → 调试 → 发布 一条直线**：用户在单一页面内完成"配置 → 调试 → 发布"，不要让用户在多页面间反复跳转。
- **市场 vs 我的** 双轨并列：左侧导航同一资产类型下，始终提供"我的 XXX"和"XXX 市场"两个入口。
- **对话为中心**：主对话页是高频页，应始终可通过固定入口一键到达。
- **渐进式表单**：复杂创建（尤其是 Agent）使用分步向导（Step 1 基本信息 → Step 2 工具 → Step 3 协作 → Step 4 调试发布），不要把所有字段堆在一页。

### 1.3 信息架构（Site Map）

```
┌─ 登录（W3 OAuth 重定向）            P01
└─ 主应用（登录后）
   ├─ 首页 / Dashboard                 P02
   ├─ 创建入口                          P03
   ├─ 我的资产
   │   ├─ 智能体
   │   │   ├─ 列表                    P04
   │   │   ├─ 创建/编辑               P05
   │   │   └─ 调试                    P06
   │   ├─ Skill
   │   │   ├─ 列表                    P09
   │   │   └─ 创建/编辑               P10
   │   ├─ MCP
   │   │   ├─ 列表                    P12
   │   │   └─ 添加                    P13
   │   └─ 知识库
   │       ├─ 列表                    P16
   │       └─ 创建/文档管理           P17
   ├─ 市场
   │   ├─ 智能体市场                  P07 / P08
   │   ├─ Skill 市场                  P11
   │   └─ MCP 市场                    P14
   ├─ 对话（Chat）                     P15
   └─ 管理后台（仅管理员）
       └─ 用户与权限                  P18
```

## 2. 全局设计规范

> 以下给出**设计锚点**，设计师可在此基础上自由发挥风格（科技蓝 / 中性灰 / 企业稳重）。

| 项目                   | 规范                                                                                     |
| ---------------------- | ---------------------------------------------------------------------------------------- |
| **设备**               | Web Desktop 优先（最小 1366 × 768），平板降级支持（768+），移动端仅保证对话页可用        |
| **主色调建议**         | 企业蓝 + 中性灰；状态色（成功/警告/错误/信息）必须与主色可视区分                         |
| **字号层级**           | H1 24 / H2 20 / H3 16 / Body 14 / Caption 12（基准字号；设计师可等比调整）               |
| **栅格**               | 12 列、外边距 24px、间距 16px                                                            |
| **圆角**               | 卡片 8px、按钮 6px、输入框 4px                                                           |
| **阴影**               | 3 级：卡片静态 / 卡片 hover / 弹层                                                       |
| **图标**               | 线性风格为主，资产类型要有专属 icon（Agent 机器人、Skill 魔法棒、MCP 插头、知识库 书本） |
| **状态色语义**         | 绿=已发布/启用；蓝=草稿/进行中；橙=调试中/需更新；红=错误/禁用；灰=未发布                |
| **空态/错误态/加载态** | 每个列表与详情页必须有三种状态稿                                                         |

## 3. 全局布局（Shell / Frame）

所有登录后页面共享如下外壳：

```
┌──────────────────────────────────────────────────────────────────┐
│  Logo │ 面包屑（当前页路径）      [搜索] [🔔 通知] [👤 用户]      │  ← 顶部栏 56px
├────────────┬─────────────────────────────────────────────────────┤
│            │                                                     │
│  侧边栏     │                主内容区                              │
│  (240px)   │                                                     │
│            │                                                     │
│  ▸ 首页     │                                                     │
│  ▸ 对话     │                                                     │
│  ─────      │                                                     │
│  [我的资产] │                                                     │
│  ▸ 智能体   │                                                     │
│  ▸ Skill    │                                                     │
│  ▸ MCP      │                                                     │
│  ▸ 知识库   │                                                     │
│  ─────      │                                                     │
│  [市场]     │                                                     │
│  ▸ 智能体   │                                                     │
│  ▸ Skill    │                                                     │
│  ▸ MCP      │                                                     │
│  ─────      │                                                     │
│  [+ 创建]   │                                                     │
│            │                                                     │
└────────────┴─────────────────────────────────────────────────────┘
```

**交互逻辑（全局）**

- 侧边栏可折叠（保留 56px 图标栏）。
- 用户头像点击 → 下拉菜单：个人资料 / 设置 / 退出登录（R9-8）。
- 顶部搜索默认搜"当前域"（如在智能体列表就搜智能体），ESC 关闭。
- 通知铃铛展示：发布成功、MCP 更新、版本更新提醒（R2-10、R6-6）。
- 右下角固定悬浮按钮 💬 快速打开对话页。
- 需求映射：R10-1, R10-2。

## P01. 登录与 OAuth 回调

### 目的

通过 W3 OAuth 完成身份认证（R9）。

### 页面布局

```
┌──────────────────────────────────────────┐
│                                          │
│            [ Logo + 平台名称 ]             │
│                                          │
│          欢迎使用 Agent Platform           │
│        一句 slogan（副标题 13px 灰）        │
│                                          │
│         ┌────────────────────────┐       │
│         │  使用 W3 账号登录  →    │       │  ← 主按钮，大号
│         └────────────────────────┘       │
│                                          │
│         登录即表示同意《使用协议》           │
│                                          │
└──────────────────────────────────────────┘
     背景：品牌渐变 / 抽象图形
```

### 交互逻辑

1. 未登录访问任何 URL → 自动重定向到本页（R9-1）。
2. 点击"使用 W3 账号登录" → 跳转 W3 OAuth 授权页。
3. 授权完成 → OAuth 回调页（白屏 + 居中 Loading："正在完成登录…"）→ 成功跳首页 P02（R9-2, R9-3）。
4. 失败态：卡片内红字提示"登录失败，点击重试"，附重试按钮（R9-4）。
5. 会话过期：静默刷新；刷新失败回到本页（R9-4）。

### 设计师需输出

- 正常态、加载态、失败态。

```
│  │                     │ │                     │ │        │ │
│  │ 描述文本最多 2 行…    │ │                     │ │        │ │
│  │                     │ │                     │ │        │ │
│  │ 🔧 工具 5 | 🧩 协作 2 │ │                     │ │        │ │

```

### 卡片规范

- 状态徽章：草稿（灰）/ 已发布（绿）/ 调试中（橙）。
- "更多"菜单：复制 / 导出 / 删除 / 发布。

### 交互逻辑

- **使用**：创建对话会话并跳 P15（R2-7 同逻辑）。
- **编辑**：跳 P05 并载入配置（R1-7）。
- **复制**：原地生成副本，新增卡片"{名称}-副本"（R1-8）。
- **导出**：直接下载 JSON 文件（R1-10）。
- **删除**：弹出确认 Dialog（标题 "确认删除"，内容"操作不可恢复"，红色主按钮），确认后列表移除（R1-9）。
- **发布**（见 P05 调试完成后）：R2-1 ~ R2-3。
- **顶部"创建智能体"**：跳 P05（R1-2）。
- **导入按钮**（"更多操作"菜单）：上传 JSON → 校验 → 生成副本（R1-11）。校验失败弹 Toast。
- 空态：居中插画 + "还没有智能体，立即创建第一个" + 主按钮。

### 设计师需输出

- 有数据态、空态、加载态、删除确认 Dialog、导入文件校验失败 Toast。

## P05. 智能体创建 / 编辑

### 目的

完成智能体完整配置（R1-2 ~ R1-5）。**采用左右分栏的分步向导**。

### 页面布局

```
┌─────────────────────────────────────────────────────────────────┐
│  < 返回    创建智能体 / 编辑智能体：{名称}        [保存草稿][发布]│
├───────────────┬─────────────────────────────────────────────────┤
│ Step 导航      │   右侧动态内容（每步不同）                        │
│ ● 1. 基本信息  │                                                 │
│ ○ 2. 工具配置  │                                                 │
│ ○ 3. 协作智能体│                                                 │
│ ○ 4. 调试发布  │                                                 │
│               │                                                 │
│ ─────────     │                                                 │
│ 保存时间：刚刚  │                                                 │
└───────────────┴─────────────────────────────────────────────────┘
```

### Step 1 — 基本信息（R1-2）

表单字段（垂直排列）：

- 头像（支持上传/选择预设）
- 名称（必填，≤30 字）
- 描述（必填，≤200 字，多行）
- 模型选择（下拉，显示模型 logo + 名称 + 简介）
- 最大步骤数（数字输入，默认 10，提示"上限保护，防止无限循环"）
- 系统提示词（Prompt，多行文本 + 变量插入按钮）
- **[下一步] 按钮** 右下

### Step 2 — 工具配置（R1-4）

```
┌─────────────────────────────────────────┐
│ 可用工具                [🔄 刷新] [搜索]  │
├─────────────────────────────────────────┤
│ ▾ 分组：内置工具                         │
│   ☐ 网络搜索      [配置参数]             │
│   ☑ 代码执行      [配置参数]             │
│ ▾ 分组：MCP-阿里云                       │
│   ☑ list_buckets                       │
│ ▾ 分组：知识库                           │
│   ☑ 检索"产品手册" [更换知识库]          │
└─────────────────────────────────────────┘
```

- 勾选 = 启用该工具。
- "配置参数"弹侧抽屉（Drawer）输入 key-value。
- 知识库检索工具可绑定具体知识库（R8-7）。

### Step 3 — 协作智能体（R1-3）

- 左：当前已选协作 Agent 列表（可拖拽排序）。
- 右：从"我的智能体 + 已收藏"中选择加入。
- 每个协作 Agent 卡片可设定"调用时机"描述（文本）。

### Step 4 — 调试发布

- 跳 P06 的内嵌形式，或按钮"进入调试 →"跳 P06。
- 调试通过后显示"发布"按钮，点击弹发布对话框（见下）。

### 发布确认 Dialog（R2-2, R2-3）

```
标题：发布智能体
──────────────────────────
 发布范围：  ◉ 公开（市场可见）
             ○ 同组可编辑
             ○ 同组只读
             ○ 私有
 版本号：    v1.0.0
 更新说明：  [多行文本]
──────────────────────────
                [取消] [确认发布]
```

- 权限 4 选项对应 R9-6。
- 已发布过再发布 → 版本号自动 +1 建议（R2-9, R2-10）。

### 交互逻辑

- 顶部右侧"保存草稿"任何步骤可点，Toast "已保存"。
- 步骤间切换不丢失未保存数据（本地暂存）。
- 离开页面前有改动未保存 → 弹"确认离开？"对话框。
- 必填校验不通过时，在字段下以红字提示。

### 设计师需输出

- 4 个 Step 的各自布局稿、发布 Dialog、字段校验报错态、工具参数 Drawer。

## 23. 关键跨页用户流程

### 流程 A：新用户首次创建并使用智能体

```
P01 登录 → P02 首页（空态）→ 点击"创建智能体"
 → P03 选"Agent" → P05 Step1~3 填写 → Step4 进入 P06 调试
 → 调试通过 → 发布 Dialog → 回到 P04 列表（新卡片"已发布"）
 → 卡片点"使用" → P15 对话
```

### 流程 B：从市场发现并使用他人智能体

```
侧栏"智能体市场" → P07 → 搜索/筛选
 → 点卡片 → P08 详情 → [使用]
 → P15 新对话 启动
```

### 流程 C：为智能体添加 MCP 工具

```
P12 MCP 列表 → 空态"去市场看看" → P14 市场
 → 找到目标 → [接入]（可能弹参数 Dialog）
 → 回 P12 测试连接 通过
 → P05 Step2 工具配置 → 勾选该 MCP 提供的工具
 → 保存 → P06 调试验证
```

### 流程 D：导入/导出智能体

```
导出：P04 卡片"更多 → 导出" → 下载 JSON
导入：P04 顶部"更多 → 导入" → 选文件
  → 校验通过：列表出现副本
  → 校验失败：Toast "文件格式不正确"
```

### 流程 E：发布与版本更新

```
首次发布：P05 → [发布] → 选公开/私有 → 输版本号/说明 → 确认
后续更新：检测到 Agent 已改动 → 顶部 Banner"有未发布更新，是否发布 v1.1.0？"
  → 用户点 Banner → 发布 Dialog（版本号自动 +1）
```

### 流程 F：会话过期处理

```
任意页面请求 401 → 静默刷新 token
  → 刷新成功：重试原请求
  → 刷新失败：全屏跳 P01 + Toast "登录已过期"
```
